Getting Started with CSS

Cascading Style Sheets (CSS) describe how elements render in the browser. Every color, layout, typeface, and animation you see on the web is ultimately driven by a cascade of CSS rules.

Rules, Selectors, and Declarations

A CSS rule matches elements with a selector and changes their presentation with declarations. Declarations are property/value pairs wrapped in curly braces.

/* A basic rule */
button.primary {
  background: #2563eb;
  color: #f8fafc;
  padding: 0.75rem 1.25rem;
  border-radius: 999px;
}

Keep related rules in dedicated files. Naming files after components—buttons.css, navigation.css, forms.css—helps you reason about growing projects.

Understanding the Cascade

The “cascade” in CSS resolves conflicts when multiple rules target the same element. The browser evaluates three pillars:

Specificity

More specific selectors win. Inline styles outrank IDs, IDs outrank classes, and classes outrank type selectors.

Source Order

If specificity is tied, the rule that appears last in the stylesheet wins. Keep overrides near the code they adjust.

Importance

!important beats specificity, but use it sparingly. Well-structured CSS rarely needs it.

Properties such as color and font-family are inherited. Layout properties like margin and padding are not. Knowing what inherits prevents unnecessary repetition.

Authoring Workflow

  • Organize modules: Split styles into focused files and import them in a predictable order.
  • Use the browser inspector: DevTools exposes the cascade, computed values, and layout overlays, which are invaluable for debugging.
  • Favor modern units: Relative units such as rem, vw, and ch adapt to user settings and different devices.
  • Keep accessibility in mind: Check color contrast, respect prefers-reduced-motion, and leverage semantic HTML before styling.

Now that you know the anatomy of a CSS rule and the rules of the cascade, continue with selectors to learn how to target elements precisely.
