CSS Selectors

Selectors are the hook between HTML and styling. They allow you to target exactly the elements you want to restyle—nothing more, nothing less.

Core Selector Types

These selectors cover most day-to-day styling needs. Combine them to describe elements unambiguously.

/* Type selector targets elements by tag name */
article { max-width: 70ch; }

/* Class selector targets elements with .warning class */
.warning { background: color-mix(in srgb, #f87171 50%, transparent); }

/* ID selector targets unique #app root */
#app { min-height: 100vh; }

Prefer classes over IDs for styling. Classes can be reused and chain together without bumping specificity unnecessarily.

Attributes and Pseudo-Classes

Attribute selectors narrow results based on the presence or value of an attribute. Pseudo-classes react to state and position.

Attribute Selectors

input[type="email"] {
  border-color: #38bdf8;
}

Use ^=, $=, or *= to match prefixes, suffixes, or substrings.

Stateful Pseudo-Classes

button:hover { transform: translateY(-1px); }
input:focus-visible { outline: 2px solid #38ef7d; }

Pseudo-classes keep interactive states accessible without JavaScript.

Structural Pseudo-Classes

li:nth-child(odd) { opacity: 0.75; }
section:last-of-type { border-bottom: none; }

Target elements based on their position in the DOM tree.

Combinators & Selector Lists

  • Descendant (A B): matches B elements anywhere inside A. Useful for utility classes applied to wrappers.
  • Child (A > B): targets direct children. Use it to avoid selecting nested descendants unintentionally.
  • Adjacent sibling (A + B): matches the first B immediately following A. Perfect for spacing content blocks.
  • General sibling (A ~ B): matches siblings that appear later in the DOM, regardless of distance.
.card > h2 + p {
  margin-top: 0.5rem;
}

Group selectors with commas to apply the same declarations to multiple targets. Next, learn how those selectors translate into real layouts in the layout guide.
