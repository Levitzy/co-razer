CSS Box Model

Every element on the page renders as a rectangular box. Understanding the layers of that box lets you control spacing with confidence.

The Four Layers

The CSS box model is composed of four concentric layers. When you add padding, borders, or margins you are growing a specific layer of the box. Adjacent vertical margins can collapse into a single value, so mind the order of your blocks.

Content
Padding
Border
Margin
  • Content is the text, image, or interactive elements inside the box.
  • Padding adds interior breathing room between content and the border.
  • Border draws a visible outline around the padding.
  • Margin creates external spacing that pushes other elements away.

Controlling Size With box-sizing

By default, width and height are applied to the content box. Switching to box-sizing: border-box; tells the browser to include padding and border in those measurements, which makes sizing more predictable.

*, *::before, *::after {
  box-sizing: border-box;
}

.card {
  width: 320px;
  padding: 1.5rem;
  border: 2px solid #38ef7d;
  margin: 1rem auto;
}

With border-box the total rendered width of .card remains 320 px even after you add padding or borders.

Debugging The Box

Browsers include tools to help you see the active box model for any element. Inspect an element and look for a layered preview of content, padding, border, and margin. When you are experimenting, temporary helper styles can also illuminate the layers:

/* Outline every layout container */
.layout > * {
  outline: 1px dashed rgba(125, 211, 252, 0.6);
  outline-offset: 4px;
}

Outlining content during development makes hidden margins or uneven padding immediately noticeable. Chrome, Edge, and Firefox DevTools also reveal when margins are collapsing by shading them with a single color.
