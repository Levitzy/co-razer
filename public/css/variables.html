CSS Custom Properties

Custom properties, also known as CSS variables, let you store reusable values and adjust them at runtime. They are scoped, cascade-aware, and incredibly powerful for theming.

Declaring Variables

Custom properties start with two hyphens. Declare globals on :root and component-level tokens inside selectors so they inherit appropriately.

--accent = #38bdf8
--surface = #0f172a
:root {
  --accent: #38bdf8;
  --surface: #0f172a;
  --radius-lg: 18px;
}

Using And Overriding

Reference variables with var(--name, fallback). Because they participate in the cascade, you can override tokens for specific themes or components.

.card {
  background: var(--surface);
  border-radius: var(--radius-lg);
  border: 1px solid color-mix(in srgb, var(--accent), transparent 70%);
}

Provide a fallback value for older browsers or unexpected overrides.

Theming Patterns

Scope variables to theme selectors and toggle them with classes or media queries. Keep semantic names (--color-text-muted) so the tokens describe intent rather than raw hues.

.theme-light {
  --color-text: #0f172a;
  --color-panel: #ffffff;
}

.theme-dark {
  --color-text: #e2e8f0;
  --color-panel: #0f172a;
}

Switching themes becomes as simple as changing a class on <body>.

Runtime Tricks And Best Practices

  • Use in calculations: Variables plug into calc(), letting you compose spacing systems such as calc(var(--gap) * 2).
  • Expose from JavaScript: Update values with element.style.setProperty('--accent', '#4ade80') to react to user preferences or live sliders.
  • Type safety: The @property rule registers custom properties with syntax and defaults, enabling transitions on color tokens and lengths.
  • Avoid hard-coding: Start with design tokens like --color-background instead of generic names so swapping palettes remains clear.

Because custom properties participate in the cascade, always define sensible defaults near the root and override them closer to the components that need them.
