CSS Typography

Typography sets the voice of your interface. Use font stacks, rhythm, and weight to create hierarchy that feels intentional.

Choosing a Font Stack

Define resilient font stacks that fall back gracefully. Modern systems ship with high quality sans-serif and serif fonts you can lean on before loading custom typefaces. Pair the stack with font-display: swap when using web fonts so text stays readable while fonts download.

Typography Preview

Readable paragraphs start with a base size around 16 px and a comfortable line height of roughly 1.5. Mixing weights draws attention without needing extra colors.

This line uses a semi-bold weight with a touch of letter-spacing for emphasis.

body {
  font-family: "Inter", "Segoe UI", Roboto, sans-serif;
  font-size: 16px;
  line-height: 1.6;
}

Scale And Rhythm

Use relative units like rem so type scales with user preferences. Fluid sizing with clamp() eases transitions between mobile and desktop, while consistent line-height keeps blocks readable. Set a comfortable measure by capping paragraph width in characters.

h1 {
  font-size: clamp(2.4rem, 5vw, 3.2rem);
  line-height: 1.1;
}

p {
  font-size: clamp(1rem, 1.2vw, 1.125rem);
  line-height: 1.6;
  max-width: 65ch;
}

Think in "characters per line": 60-75 characters maintains flow for long-form content.

Weight, Style, And Letter-Spacing

Adjust font-weight and font-style to create a visual hierarchy. For uppercase or wide headings, add gentle letter-spacing to prevent crowding. Variable fonts expose finer weight granularity; when available, choose numeric weights (e.g. font-weight: 480;).

.eyebrow {
  text-transform: uppercase;
  font-weight: 600;
  letter-spacing: 0.08em;
  color: rgba(125, 211, 252, 0.9);
}

Limit the number of distinct font sizes on a page to keep typography cohesive. Lean on CSS custom properties to describe typography scales so updates remain consistent.

Readability And Accessibility

  • Contrast: Meet WCAG contrast ratios (4.5:1 for body copy, 3:1 for large text). Tools like DevTools' color picker confirm compliance.
  • Spacing: Give interactive elements enough line-height and padding so they remain tappable on touch devices.
  • Motion preferences: Honor @media (prefers-reduced-motion) by disabling elaborate text animations for users who request it.
  • Language support: When serving multilingual sites, ensure your font stack includes fonts that cover required glyphs or defines fallbacks per writing system.

Thoughtful typography respects user settings, devices, and accessibility needs as much as visual polish.
