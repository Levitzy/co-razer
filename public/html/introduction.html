HTML: Introduction

HTML (HyperText Markup Language) is the foundation of web development. Learn how to create semantic, accessible, and modern web pages using HTML5 elements and best practices.

What Is HTML?

  • Defines the structure and meaning (semantics) of content.
  • Consists of elements written with tags like <p> and <a>.
  • Browsers parse HTML and render the content; tags themselves are not shown.

Basic Page Structure

<!DOCTYPE html>
  <html lang="en">
    <head>
      <meta charset="utf-8" />
      <meta name="viewport" content="width=device-width, initial-scale=1" />
      <title>Hello</title>
    </head>
    <body>
      <h1>Hello, Co.Razer!</h1>
      <p>This is your first HTML page.</p>
    </body>
  </html>

Example explained:

  • <!DOCTYPE html> declares HTML5.
  • <html> is the root element; lang helps accessibility/SEO.
  • <head> contains metadata (not shown on the page), including <title>.
  • <body> contains the visible content.

Elements and Tags

An HTML element usually has an opening tag, content, and a closing tag:

<p>This is a paragraph.</p>
  <a href="#">This is a link</a>
  • Void elements have no closing tag (e.g., <img>, <br>, <meta>).
  • Nesting matters: elements must be properly nested and closed.
  • Block vs. inline: block elements (e.g., <div>, <section>) start on a new line; inline elements (e.g., <span>, <a>) flow within a line.

Attributes

Attributes provide extra information about elements. They go inside the start tag as name="value".

<a href="https://example.com" title="Visit example">Example Link</a>
  <img src="/assets/logo.png" alt="Co.Razer logo" width="120" />
  • href for links, src/alt for images.
  • id (unique) and class (reusable) for styling and scripting.
  • lang on <html> and title on many elements improve accessibility.

Semantic HTML

Use semantic elements to describe purpose, not appearance. This improves accessibility and SEO.

<header>...site header...</header>
  <nav>...main navigation...</nav>
  <main>
    <article>
      <h1>Article title</h1>
      <section>Content section</section>
    </article>
  </main>
  <footer>...site footer...</footer>

Comments

<!-- This is a comment. It won't be rendered. -->

Best Practices

  • Use lowercase tag/attribute names and quote attribute values.
  • Always include alt text for images and labels for form inputs.
  • Keep structure semantic; avoid using HTML for visual layout (use CSS).
  • Set <meta charset="utf-8"> and a responsive viewport.

Head vs. Body

The <head> contains metadata and links to resources; the <body> contains what users see and interact with.

<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Page title</title>
    <link rel="stylesheet" href="/assets/styles.css">
    <script defer src="/assets/app.js"></script>
  </head>
  <body>
    <main>Visible content here</main>
  </body>

File Paths

Use correct paths when linking to other files. A path starting with / is from the site root; otherwise it’s relative to the current file.

<!-- Absolute-from-root path -->
  <img src="/assets/logo.png" alt="Logo">

  <!-- Relative to current folder -->
  <img src="images/photo.jpg" alt="Photo">

  <!-- Go up one folder -->
  <link rel="stylesheet" href="../shared/base.css">

Character Entities

Some characters are reserved in HTML and must be escaped using entities:

&lt;  &mdash;  &copy;  &amp;
  <p>Use &amp;lt; to display a literal &lt; sign.</p>

Accessibility Basics

  • Provide meaningful alt text for images.
  • Use a logical heading order (h1 → h2 → h3...).
  • Associate form inputs with <label> elements.
  • Use semantic elements (<nav>, <main>, <footer>) for structure.

Validate and Debug

  • Validate markup with the W3C HTML Validator (validator.w3.org).
  • Use browser DevTools (Elements/Inspector) to inspect and edit the DOM.
